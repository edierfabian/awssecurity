    AWS Firewall Manager simplifica la administración y el mantenimiento de sus grupos de seguridad de AWS WAF, AWS Shield Advanced y Amazon VPC en múltiples cuentas y recursos. Con Firewall Manager, solo tiene que configurar las reglas de firewall de AWS WAF, las protecciones de Shield Advanced y los grupos de seguridad de Amazon VPC una sola vez. El servicio aplica automáticamente las reglas y protecciones en sus cuentas y recursos, incluso al añadir nuevos recursos.

    Firewall Manager ofrece estos beneficios:

    – Ayuda a proteger los recursos en todas las cuentas

    Categoría: SEC – Seguridad de infraestructura

    Una empresa planea alojar un portal de comercio electrónico en contenedores en varios clústeres de Amazon ECS, que se lanzan en varias cuentas de AWS. Las credenciales de la base de datos deben proporcionarse a la AMI de comercio electrónico mediante variables de entorno. Se encargó a un ingeniero de seguridad que garantizara la seguridad de las credenciales al transferirlas a la imagen y que no se pudieran ver en el propio clúster. Las credenciales deben almacenarse en un almacenamiento dedicado con gestión del ciclo de vida y rotación de claves. También debe implementarse un sistema automatizado que evalúe el cumplimiento de la seguridad de las credenciales en todas las cuentas de su organización de AWS.

    ¿Cuál de las siguientes es la solución más adecuada en este escenario que el ingeniero puede implementar con el menor esfuerzo?

    • 1.
    • 2.
    • 3.
    • 4.

    Correcto

    Amazon ECS le permite inyectar datos confidenciales en sus contenedores almacenándolos en secretos de AWS Secrets Manager o en parámetros del almacén de parámetros de AWS Systems Manager y haciendo referencia a ellos en la definición del contenedor. Esta función es compatible con tareas que utilizan los tipos de lanzamiento EC2 y Fargate.

    En la definición del contenedor, especifique  secrets el nombre de la variable de entorno que se establecerá en el contenedor y el ARN completo del secreto de Secrets Manager o del parámetro del almacén de parámetros de Systems Manager que contiene los datos confidenciales que se presentarán al contenedor. El parámetro al que haga referencia puede ser de una región diferente a la del contenedor que lo utiliza, pero debe pertenecer a la misma cuenta.

    AWS Secrets Manager es un servicio de gestión de secretos que le ayuda a proteger el acceso a sus aplicaciones, servicios y recursos de TI. Este servicio le permite rotar, administrar y recuperar fácilmente credenciales de bases de datos, claves de API y otros secretos a lo largo de su ciclo de vida. Con Secrets Manager, puede proteger y administrar los secretos utilizados para acceder a recursos en la nube de AWS, en servicios de terceros y en las instalaciones locales.

    Puede configurar el Agregador de Datos Multicuenta y Multiregión de AWS Config para revisar las configuraciones de sus secretos en todas las cuentas y regiones de su organización y, a continuación, revisarlas y compararlas con las prácticas recomendadas de administración de secretos. Debe habilitar AWS Config y las reglas administradas de AWS Config específicas para los secretos en todas las cuentas y regiones antes de crear un agregador.

    Si desea un único almacén para la configuración y los secretos, puede usar el Almacén de Parámetros. Si desea un almacén de secretos dedicado con gestión del ciclo de vida, use el Administrador de Secretos.

    Por lo tanto, la respuesta correcta es la opción que indica: Usar AWS Secrets Manager para almacenar las credenciales de la base de datos y luego cifrarlas con AWS KMS. Configure un rol de IAM para su rol de ejecución de tareas de Amazon ECS y referenciarlo con su definición de tarea, lo que permite el acceso tanto a KMS como a AWS Secrets Manager. En la definición de su contenedor, especifique los secretos con el nombre de la variable de entorno que se establecerá en el contenedor y el ARN completo del secreto de Secrets Manager que contiene los datos confidenciales para presentarlos al contenedor. Configure el Agregador de Datos Multi-Cuenta y Multi-Región de AWS Config para revisar la configuración de las credenciales en todas las cuentas.

    La opción que dice: Usar AWS Systems Manager Parameter Store para almacenar y administrar las credenciales de la base de datos y luego cifrarlas usando AWS KMS. Configure un rol de IAM para su rol de ejecución de tareas de Amazon ECS y haga referencia a él con su definición de tarea, lo que permite el acceso tanto a KMS como al Parameter Store. Dentro de su definición de contenedor, especifique secretos con el nombre de la variable de entorno que se establecerá en el contenedor y el ARN completo del parámetro Systems Manager Parameter Store que contiene los datos confidenciales que se presentarán al contenedor. Configurar AWS Config en cualquier cuenta de AWS miembro de la AWS Organization para verificar el cumplimiento de seguridad de las credenciales en todas las cuentas  es incorrecto. Aunque el uso de Systems Manager Parameter Store para proteger datos confidenciales en ECS es válido, este servicio no proporciona almacenamiento dedicado con administración del ciclo de vida y rotación de claves, a diferencia de Secrets Manager. Además, el servicio AWS Config no puede verificar los secretos alojados en AWS Secrets Manager de otra cuenta de AWS de forma predeterminada. Debe configurar el Agregador de datos multicuenta y multiregión de AWS Config para revisar las configuraciones de las credenciales en todas las cuentas.

    La opción que dice: Almacenar las credenciales de la base de datos y cifrarlas con KMS, luego referenciarlas en el archivo de definición de tarea de ECS del clúster de ECS. Almacenar el archivo JSON de definición de tarea en AWS Control Tower. Configurar un rol de IAM para el script de definición de tarea de ECS que permita el acceso a AWS Control Tower para la register-task-definitionacción de ECS. Referenciar el archivo JSON de definición de tarea en Control Tower que contiene las credenciales de la base de datos. Configurar AWS Config en cualquier cuenta de AWS miembro de la organización de AWS para verificar el cumplimiento de seguridad de las credenciales en todas las cuentas de AWS es incorrecto porque, dado que no puede almacenar credenciales de base de datos, ni ningún otro tipo de datos, en el servicio AWS Control Tower. Se utiliza principalmente para establecer una zona de aterrizaje para su organización de AWS y simplifica la gobernanza de sus cuentas de AWS mediante barandillas. Además, debe configurar el agregador de datos multicuenta y multiregión de AWS Config para este escenario.

    La opción que dice: Almacenar las credenciales de la base de datos con AWS Security Hub para administrar centralmente los datos confidenciales y transmitirlos de forma segura solo a los contenedores que necesitan acceder a ellos. Consulte las credenciales en el archivo de definición de tareas de ECS del clúster de ECS. Los secretos se cifran automáticamente durante el tránsito y en reposo. Configurar el Agregador de Datos Multi-Cuenta y Multi-Región de AWS Config para revisar la configuración de las credenciales en todas las cuentas es incorrecto. El servicio AWS Security Hub simplemente recopila datos de seguridad de todas las cuentas, servicios y productos de socios externos compatibles de AWS para ayudarle a analizar sus tendencias de seguridad e identificar los problemas de seguridad más prioritarios. Este servicio no puede almacenar credenciales de seguridad de ningún tipo. En AWS, la forma recomendada de proteger los datos confidenciales es mediante el uso de Secrets Manager o Systems Manager Parameter Store.
